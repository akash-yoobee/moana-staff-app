Attributions

To finish this project a lot of resources were used that were not created by myself. Many individuals and companies allowed their work to be used freely.


The website "Unsplash" (https://unsplash.com) allowed pictures uploaded by photographers to be used without individually attributing every picture to it's photographer. The following were used to make the food ordering software.

  • • Five pictures for making user profiles.
  • • One background image for making the side-navbar for the staff facing app.
  • • Five pictures of burgers for making the food menu.
  • • Five pictures of chicken dishes for making the food menu.
  • • Four pictures of desserts for making the food menu.
  • • Four pictures of drinks for making the food menu.
  • • Four pictures of food derived from fish for making the food menu.
  • • Five pictures of sides for making the food menu.
  • • Four pictures for making page header cards for the client facing app.
  • • Four pictures for illustrating the user experience report (UX report).

Two pictures of real people from the 2019 class studying the "Diploma in Web and UX Design" at Yoobee were used. These were retrieved from the internet (RocketReach, n.d.) (Lewis, 2019).

Some free to use fonts from the website "Font Awesome" (https://fontawesome.com/) were used as icons in the software.

Some icons made by Google which they made available freely to web developers were used throughout the software (Google, n.d.).

The following fonts were used throughout the software and the UX report

  • • Tribal Case (Huse, n.d.)
  • • Quesha (NimaVisual, n.d.)
  • • Rockwell (Pierpont, n.d.)
  • • Akzidenz-Grotesk (Berthold, 1898)
  • • Poppins a Google font designed by "Indian Typefoundry" (Indian Typefoundry, n.d.)
  • • Ink Free (Matteson, n.d.)

CKEditor version 4.8.0 (CKSource, n.d.) was used to enable the users of the staff facing app to enter data about food items and the shop's employees.

Chart.js version 2.8.0 (https://www.chartjs.org) an open source chart making framework was used to display data as a chart in the dashboard of the staff facing app.

Materialize.css version 0.100.2 (https://materializecss.com) a responsive front-end framework was used to develop the software.

JQuery version 3.2.1 (https://jquery.com/) a JavaScript library was used to develop the software.

Ayoa a visual design production software was used to represent the software's content strategy and ideation process in the UX report.

Marvel (https://marvelapp.com/) an on-line design platform for prototyping software was used to create low fidelity prototypes for inclusion in the UX report.

Figma (https://www.figma.com/) an on-line design platform for prototyping software was used to create high fidelity prototypes for inclusion in the UX report.

Microsoft word (https://www.office.com/launch/word) was used to create the UX report.

Visual Studio Code (https://code.visualstudio.com/) was used to develop code for the software.

GitHub (https://github.com/) was used to aid in the software development process.

A video from the Anglo-link YouTube channel was used for guidance on using the passive voice in English (Anglo-Link, 2013).

The websites of BurgerFuel (https://www.burgerfuel.com/nz), Burger King (https://www.burgerking.co.nz/) and McDonalds (https://mcdonalds.co.nz/) were used for competitor analysis during the software development process.

Interviews with Richard Lovell (R. Lovell, personal communication, 9 September 2019), Joel Innes (J. Innes, personal communication, 12 September 2019) and Amber Page (A. Page, personal communication, 12 September 2019) were used to guide the software development process.

Educational resources on Yoobee Colleges's (https://www.yoobee.ac.nz) website were used to learn about and guide the software development process.
